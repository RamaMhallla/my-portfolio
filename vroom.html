VROOM — Backend Dashboard & APIs (Laravel)

Role: Backend Developer (Laravel)

Company engagement building a production logistics platform: admin dashboard modules and public application APIs.

Source Code Confidentiality: Developed under a client contract/NDA. The source code is proprietary and not available for distribution. This page demonstrates functionality through interface screenshots.

Project Overview

VROOM is a delivery and logistics platform serving B2C and B2B flows. The Laravel backend powers secure REST APIs and an administration dashboard for orders, customers, payments, vehicles, zones, roles/permissions, promo codes, notifications, and operational settings (working hours, capacities, fees).

Technical Summary

  • Designed and implemented RESTful APIs for orders, users, payments, promos, and configuration modules.
  • Built RBAC (SuperAdmin, managers, clients) with guarded routes, policies, and middleware.
  • Integrated multiple payment options (Online Payment, Apple Pay, Wallet, Cash on Delivery) incl. webhooks and idempotency.
  • Developed admin analytics & reporting endpoints (KPIs, vehicle mix, B2B/B2C trends, driver performance).
  • Implemented promo engine (discount, usage caps, validity windows, status lifecycle).
  • Operational controls: vehicles, zones, fees, working/blocking hours, wallet/B2B packages.
  • Performance: query optimization (indexes, eager loading), caching/queues (Redis) for reliability at load.
  • Quality: request validation, structured error handling, environment-based config, OpenAPI/Swagger docs.
